 -기업 망에서 구축되는 LAN 구성도
 -소프트웨어가 탑재된 하나의 서버와 장비

2. 네트워크 구성요소

 -server   : 요청한 서비스를 처리해주는 역활의 시스템
 -client   : 서비스를 요청하는 역활을 가진 시스템
 -media    : 서버와 클라이언트를 연결시켜주는 다양한 장비
 -protocol : 약속된 같은 언어로 대화할 수 있도록 제공하는 통신규약
 -service  : 사용자들이 요구하는 다양한 application 메세지

 
-컴퓨터 네트워크 구성-

server: 윈도우는 기본적으로 MS공유와 연결 서비스를 이용하여 파일과 프린터 자원을 연결할 수 있음

protocol : TCP/IP를 사용한 논리적인 주소인 IP Address를 이용한 통신 protocol

NIC : Ethernet카드를 사용한 물리적인 주소인 MAC Address를 이용한 인접한 네트워크 노드 간의 통신 전송 매체
(1) 네트워크 인터페이스 카드라고 하고, 보통 LAN Card라고 부름
(2) 컴퓨터와 케이블을 연결해주는 장치
(3) 컴퓨터신호를(네트워크 케이블로 보내기위해) 전기신호로 바꾸고 다시 (케이블을 통해 전달되는 전기신호)컴퓨터신호로 바꾸는 역할
(4) 네트워크 구축 기술에 따라 다양한 NIC가 존재

IP Address
(1) 인터넷 표준 Protocol인 TCP/IP 내부 네트워크 도입하여 인트라넷 환영에서 사용
(2) 논리적인 주소인 IP Address 사용
(3) IP Address 32비트 10진수로 표시
(4) 고유의 IP Address 가지고 있어야 통신 가능

tracert : 원격 호스트 연결 시 라우터 경로 표시
(1) -d : 주소를 호스트 이름으로 확인하지 않음
(2) -w timeout : 각 응답 대기 시간 제한 (밀리초)
(3) -R : 왕복 경로 추적 (IPv6에만 해당) 

route print: 로컬 컴퓨터 Routing Table 표시
(1) -d : 모든 게이트웨이 항목의 라우팅 테이블 지움

netstat:Protocol 통계와 현재 TCP/IP 네트워크 연결 표시
(1) -e : 이더넷 통계 표시,-s 옵션과 함께 사용할 수 있음
(2) -s : Protocol별 통계 표시
(3) -a : 모든 연결과 수신 대기 포트 표시
(4) -b : 활성화된 연결 세션과 실행한 프로그램 확인
(5) -o : 각 연결의 소유자 프로세스 ID 표시
(6) -n : 주소와 포트 번호 숫자 형식으로 표시
